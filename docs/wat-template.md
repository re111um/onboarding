## W — Workflows (작업 흐름)

> 코드/스킬을 만들기 전에 작업 흐름을 글로 먼저 정의한다.
> 10분 투자로 수시간의 삽질을 줄인다.

### 전체 흐름

```

4. 각 단계마다 검증
```

## A — Agents (에이전트 구성)

> 어떤 에이전트가 어떤 역할을 맡는지 정의한다.
> 독립적인 작업은 Sub-Agent로 병렬 처리한다.

### 메인 에이전트

- **역할**: 전체 조율 + 결과 통합
- **모델**: (opus / sonnet / haiku)

### Sub-Agent 구성

| 에이전트 | 역할 | 모델 | 도구 권한 |
|----------|------|------|-----------|
| | | | |
| | | | |

### 에이전트 간 관계

```
메인 → Sub-Agent A (독립 작업)
     → Sub-Agent B (독립 작업)
     → 결과 통합 → 다음 단계
```
