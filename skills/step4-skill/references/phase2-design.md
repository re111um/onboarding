# Phase 2: 스킬 설계 — 문제 정의 + 입출력 + 단계

## EXPLAIN

### 스킬 설계 3단계

좋은 스킬을 만들려면 코드를 쓰기 전에 설계부터 해야 한다. 3단계로 나눠보자.

**1단계: 문제 정의 — 어떤 반복 업무를 자동화하고 싶은가?**

핵심 질문: "이 작업을 매번 할 때마다 귀찮다고 느끼는가?" 또는 "이 작업에서 실수가 반복되는가?"
자동화할 가치가 있는 업무는 대부분 이 두 질문 중 하나에 "예"라고 답한다.

**2단계: 입출력 정의 — 무엇을 입력하면 무엇이 나오는가?**

- 입력(트리거): `/스킬이름` + 추가 정보 (고객사명, 사이트 ID, 프로젝트명 등)
- 출력(결과): 문서, 메시지, 데이터 요약 등

**3단계: 단계 설계 — 어떤 순서로 실행되는가?**

- Step 1에서 무엇을 하고
- Step 2에서 무엇을 하고
- 최종적으로 어떤 산출물이 나오는지

### 팀별 후보 아이디어

영감이 필요하다면 팀에 맞는 아이디어를 참고하세요:

**세일즈팀**
- 고객사 브리핑 생성기: 고객사 ID 입력 -> 관리 도구 + 시트 + Notion 데이터를 모아서 1장 요약
- 미팅 준비 도우미: 고객사명 입력 -> 미팅 로그 + 거래액 + 컨택 이력 정리
- 일일 리드 대시보드: 실행하면 -> 오늘 할 컨택, 미팅, 팔로업을 한눈에

**개발팀**
- PR 리뷰 요약기: PR 번호 입력 -> 변경 사항 + 리뷰 코멘트 + 체크리스트 정리
- 배포 상태 체크: 실행하면 -> 현재 스테이징/프로덕션 상태 + 최근 배포 이력 요약
- 코드 품질 리포트: 파일/디렉토리 입력 -> 코드 스멜, 중복, 테스트 커버리지 분석

**제품팀**
- 유저 피드백 수집기: 채널 ID 입력 -> Slack 피드백 채널에서 최근 N일 피드백 정리
- 스프린트 요약 생성기: 실행하면 -> 이번 스프린트 진행률 + 블로커 + 주요 변경사항
- 경쟁사 업데이트 모니터: 경쟁사명 입력 -> 최근 뉴스 + 기능 업데이트 정리

**경영관리팀 (HR/재무)**
- 월간 인원 현황 리포트: 실행하면 -> 입퇴사 현황 + 조직도 변경사항 + 헤드카운트 요약
- 비용 정산 체크리스트: 월말에 실행 -> 정산 필요 항목 + 증빙 누락 체크 + 마감 리마인더
- 온보딩 진행 상태 트래커: 신규입사자명 입력 -> 온보딩 체크리스트 진행률 + 남은 할일 정리

**CX팀**
- 고객 문의 유형별 주간 리포트: 실행하면 -> 이번 주 문의 유형 분류 + 빈도 + 트렌드
- FAQ 자동 업데이트 도우미: 기간 입력 -> 반복 문의에서 FAQ 후보 추출 + 초안 작성
- VOC 트렌드 분석기: 실행하면 -> 최근 N일 고객 피드백 키워드 + 감성 분석 요약

어떤 아이디어든 좋다. 중요한 건 **본인이 실제로 쓸 스킬**을 만드는 것이다.

## EXECUTE

다음 순서대로 진행하세요:

**1. 브레인스토밍**
Claude에게 이렇게 물어보세요:

**2. 아이디어 1개 선정**
위 후보 아이디어 또는 브레인스토밍 결과에서 하나를 고르세요.
선정 기준: "지금 당장 있으면 편하겠다" 싶은 것.

**3. SKILL.md 초안 작성 시작**
Claude와 대화하면서 다음을 채워보세요:
- frontmatter: name, description
- 이 스킬이 해결하는 문제 한 줄
- 입력과 출력
- 실행 단계 (Step 1, 2, 3...)

아직 완성하지 않아도 된다. 초안 수준이면 충분하다 — Phase 3에서 완성한다.
