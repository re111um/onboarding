# Phase 1: 실서비스 참관 준비

## EXPLAIN

### 실서비스 참관, 왜 해야 하나요?

{{COMPANY_NAME}} 구성원이라면 **실서비스가 실제로 동작하는 모습을 꼭 한번은 직접 관찰**해야 합니다.

실서비스를 직접 관찰하면서:

- 우리 고객은 어떻게 서비스를 이용하는지
- 서비스 제공자와 이용자는 어떤 flow로 서비스에 참여하는지
- 실제 현장에서 어떤 이슈가 발생하는지

를 직접 눈으로 확인하는 과정입니다.

Step 2에서 서비스 제공자/이용자 관점을 체험하고, Step 3에서 BM/고객 여정을 학습했는데,
이제 **실제 현장에서 그것이 어떻게 동작하는지** 확인하는 겁니다.

### 참관 당일/익일 근무 안내

> 아래 내용을 자사 정책에 맞게 수정하세요.

- **당일 + 익일**: 재택근무 + 유연근무 가능 (별도 증빙 불필요)
- **초과근무 시간**: 다음 날 근무시간에서 차감
- **팀 내 공유 필수**: 참관 일정과 다음 날 유연근무 계획을 팀 채널에 미리 공유

## EXECUTE

> **에이전트 지시 -- Notion 원본 문서 안내**:
> 1. `notion-fetch`로 실서비스 참관 가이드 페이지(`step6_live_audit` ID)를 fetch한다
> 2. fetch된 내용에서 참관 준비, 체크리스트, 과거 사례 등 보충 정보를 안내한다
> 3. Notion 원본 링크도 함께 공유
> 4. fetch 실패 시: 아래 내용만으로 충분히 안내 가능 (fallback)

### 1. 참관 대상 확인

> 자사 서비스에 맞는 참관 대상 확인 방법을 작성하세요.
> (예: 백오피스에서 일정 확인, 관리자 페이지에서 조회 등)

참관 대상 선정이 어려우면 HR({{HR_LEAD}})에게 문의하세요.

### 2. 참관 전 준비

1. 참관 관련 Slack 채널에 가입
2. 참관할 일정을 **팀 채널에 공유** (당일/익일 유연근무 예정도 함께 안내)
3. Notion의 참관 체크리스트를 확인

### 3. 실제 참관

참관 당일:
1. 서비스 시작 **15분 전**에 접속
2. 모니터링 채널에 참관 중임을 알림
3. 장애 확인 시 신속하게 리포팅
4. 아래 관점으로 관찰하며 메모:
   - 이용자 유입 flow
   - 서비스 제공자의 핵심 전략
   - 서비스 UX에서 느낀 점

> 참관이 끝나면 쉬시고, `/onboarding 이어하기`로 돌아와서 Phase 2를 진행해주세요!

### 4. 자동 리마인더 설정

에이전트가 Phase 1 CHECK 완료 시 자동으로 수행:

1. **참관 날짜를 확인**한다 (신규입사자에게 "참관이 몇 월 며칠인가요?" 질문)
2. **참관 익일 오후 2시**에 신규입사자에게 Slack 예약 메시지 발송:
   > "어제 참관 수고 많았어요! 시간 될 때 `/onboarding 이어하기`로 돌아와서 모니터링 리포트를 작성해주세요."
3. **HR({{HR_LEAD}})에게도 DM 발송**:
   > ":eyes: *실서비스 참관 알림*\n{닉네임} ({직무}, {팀})이 {날짜} 실서비스 참관 예정입니다."
4. **리포트 미작성 팔로업**: 참관 **3영업일 후**에도 리포트 미작성 시 HR({{HR_LEAD}})에게 추가 알림

> 리마인더 발송 도구: `slack_schedule_message` MCP 사용
