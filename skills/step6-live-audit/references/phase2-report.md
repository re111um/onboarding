# Phase 2: 실서비스 모니터링 리포트

## EXPLAIN

### 참관 경험을 리포트로 남기는 이유

실서비스 참관은 "봤다"로 끝나면 절반만 한 것입니다.
관찰한 내용을 **구조화된 리포트**로 정리하면:

- 본인이 무엇을 관찰했는지 명확하게 인지할 수 있고
- 팀에 공유할 때 실질적인 인사이트가 전달되며
- 나중에 다시 참관할 때 비교 기준이 됩니다

Step 2에서 체험한 서비스 제공자/이용자 관점, Step 3에서 학습한 BM/고객 여정이
**실제 서비스에서 어떻게 동작했는지** 연결 지어 정리하는 것이 핵심입니다.

### 관찰 관점

> 자사 서비스에 맞게 관찰 관점을 수정하세요.

리포트를 작성할 때 아래 관점을 기준으로 정리합니다:

**1. 이용자 유입 flow**
- 이용자가 어떤 경로로 서비스에 들어왔는지
- 참여 신청에서 실제 이용까지의 이탈 구간이 있었는지

**2. 서비스 제공자의 핵심 전략**
- 서비스 제공자가 어떤 전략으로 고객을 유치/전환했는지
- 이용자 반응은 어땠는지

**3. 서비스 UX**
- 이용자 입장에서의 UX 흐름
- 좋았던 점 (Good) / 아쉬웠던 점 (Bad)
- 장애나 불편 사항이 있었는지

## EXECUTE

> **에이전트 지시 -- 리포트 작성 지원**:
> 1. 신규입사자에게 위 템플릿을 제공한다
> 2. "참관에서 가장 인상 깊었던 장면이 뭐였어요?" 등 대화를 통해 내용을 끌어낸다
> 3. 신규입사자가 답변하면 템플릿에 맞춰 리포트를 구성해준다
> 4. 완성된 리포트를 Notion에 페이지로 생성한다 (notion-create-pages MCP 사용)
> 5. Notion 생성 실패 시 fallback: `outputs/{닉네임}-{입사일}/step6-live-audit-report.md`로 로컬 저장

### 작성 가이드

1. **메모를 꺼내세요**: 참관 중 메모한 내용을 기반으로 작성합니다. 메모가 부족해도 괜찮아요 -- 기억나는 것부터 적으면 됩니다.
2. **관점을 하나씩 채우세요**: 모든 항목을 완벽하게 채울 필요는 없습니다. 관찰한 것 위주로 솔직하게 작성하면 충분합니다.
3. **총평을 꼭 작성하세요**: "가장 인상 깊었던 점"과 "내 직무에 적용할 인사이트"가 리포트의 핵심입니다.
4. Claude에게 "리포트 작성 도와줘"라고 하면 대화를 통해 내용을 함께 채울 수 있어요.
