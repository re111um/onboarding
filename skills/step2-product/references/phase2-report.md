# Phase 2: AI 제품 개선안 (선택)

## EXPLAIN

### AI가 개밥먹기를 분석해줄 수 있어요

Phase 1에서 서비스 제공자/이용자 체험을 하면서 개밥먹기 노션 문서에 기록하셨죠?

여기서 한 가지 **보너스 체험**을 해볼 수 있어요:
Claude의 서브에이전트 3개(UX, 기술, 비즈니스)가 여러분의 개밥먹기 문서를 **동시에 분석**해서, 각 관점의 제품 개선안을 만들어주는 기능이에요.

이건 필수가 아니에요! 하지만 AI가 여러 관점에서 문서를 분석하는 과정을 체험해보고 싶다면 해볼 수 있어요.

완성되면 개밥먹기 노션 문서 안에 **AI 제품 개선안**이 하위 페이지로 생성됩니다:

```
📄 개밥먹기 — {닉네임}              ← 여러분이 작성 (Phase 1)
  └─ 📄 AI 제품 개선안            ← Claude가 자동 생성
```

---

## EXECUTE

> 이번 Phase에서는 🖥️ **Claude Code 터미널**을 사용합니다.

> **에이전트 지시** (신규입사자에게 보여주지 않는 내부 메모):
>
> **이 Phase는 선택형이다.**
> 1. 개밥먹기 문서 작성이 완료되었는지 확인한다
> 2. AskUserQuestion으로 묻는다:
>    "개밥먹기 문서 작성은 잘 마무리하셨나요?
>    여기서 **보너스 체험**을 해볼 수 있어요 — AI 서브에이전트 3개(UX/기술/비즈니스)가
>    여러분의 개밥먹기 문서를 동시에 분석해서 제품 개선안을 만들어줍니다.
>    체험해보시겠어요? (해볼게요 / 다음 단계로 넘어갈게요)"
> 3. **"해볼게요"** → 3-A로 진행
> 4. **"넘어갈게요"** → Phase 2 스킵, progress.json 업데이트 후 Step 2 마무리로
>
> **3-A: 개밥먹기 문서 확인**
> 1. 신규입사자의 개밥먹기 노션 문서 페이지를 Notion에서 찾는다
>    - `notion-search`로 신규입사자 닉네임 + "개밥먹기" 키워드로 검색
>    - 또는 `onboarding_db`에서 해당 카드를 찾는다
> 2. `notion-fetch`로 문서 내용을 읽는다
> 3. 문서가 비어 있거나 내용이 너무 적으면:
>    - "체험하면서 느낀 점을 조금 더 적어주시면 AI 개선안의 품질이 올라가요!" 안내
>    - 작성을 기다린다 → STOP
> 4. 내용이 충분하면 바로 3-B로 진행한다
>
> **3-B: AI 개선안 생성 (병렬 분석)**
>
> **⚠️ 중복 감지 (생성 전 필수 확인)**
> 1. 개밥먹기 노션 문서의 하위 페이지에서 기존 AI 개선안이 있는지 확인한다:
>    - `notion-search`로 "AI 개선안" 키워드로 검색한다
>    - 검색 결과가 없으면 "AI 제품 개선안" 키워드로 한 번 더 검색한다
>    - 검색 결과 중 해당 개밥먹기 문서의 하위 페이지인 것만 필터링한다
> 2. **이미 존재하는 경우** → AskUserQuestion으로 묻는다:
>    "이전에 작성한 AI 개선안이 이미 있습니다. 어떻게 할까요?
>    (새로 작성할게요 → 기존 페이지 삭제 후 새로 생성 / 기존 것을 업데이트할게요 → 기존 페이지에 덮어쓰기)"
>    - **"새로 작성"** → 기존 페이지를 아카이브(`notion-update-page`로 archived: true)한 뒤 아래 1번부터 진행
>    - **"업데이트"** → 아래 1~2번으로 분석 결과를 만든 뒤, `notion-update-page`로 기존 페이지 내용을 덮어쓴다 (새 페이지 생성 안 함)
> 3. **존재하지 않는 경우** → 아래 1번부터 정상 진행
>
> 1. Agent 도구로 3개 서브에이전트를 **동시에** 실행한다:
>    - **UX/사용성 에이전트**: 사용자 경험, UI 흐름, 직관성, 접근성 관점 분석
>    - **기능/기술 에이전트**: 기능 완성도, 성능, 안정성, 기술적 개선 관점 분석
>    - **비즈니스 에이전트**: 수익화, 전환율, 리텐션, 경쟁력 관점 분석
>    - 각 에이전트에게 개밥먹기 문서 전문을 전달하고, 해당 관점의 개선 제안 3~5개를 요청한다
> 2. 3개 에이전트 결과를 통합한다:
>    - 관점 간 중복 제거 + 유사 제안 병합
>    - 영향도/난이도 기준 우선순위 정리
>    - 아래 "AI 개선안 구조"에 맞춰 최종 문서 작성
> 3. AI 개선안을 **개밥먹기 노션 문서 페이지의 하위 페이지**로 생성한다:
>    - `notion-create-pages`의 `parent` 파라미터에 개밥먹기 노션 문서 페이지 ID를 지정
>    - ❌ 온보딩 칸반과 별도로 생성하지 않는다 — 반드시 개밥먹기 노션 문서 아래에 넣는다
> 4. Notion 생성 실패 시 fallback: 로컬 `outputs/` 폴더에 저장하고 "나중에 Notion에 옮기면 됩니다" 안내
> 5. progress.json의 `step2_status`를 `"completed"`로 설정
> 6. 완료 후 신규입사자에게 안내한다:
>    - "{닉네임}이 작성하신 개밥먹기 문서 하단에 AI 개선안을 작성했어요!" 메시지
>    - 생성된 페이지의 Notion URL 공유
